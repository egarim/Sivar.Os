# Sentiment Analysis Feature Enhancements

This issue tracks potential enhancements to the sentiment analysis system beyond the core implementation completed in the sentiment branch.

## 🎨 UI-Based Features

### 1. Display Sentiment in Post Cards (High Priority - ~1 hour)
- Add sentiment badges to PostCard component (Joy 😊, Sadness 😢, Anger 😠, Fear 😨, Neutral 😐)
- Display emotion score and polarity bar
- Show moderation flags (HasAnger, NeedsReview indicators)
- Validate end-to-end feature visibility

### 2. Sentiment Analytics Dashboard (Medium Priority - ~2-3 hours)
- Create emotion distribution charts (pie chart of Joy/Sadness/Anger/Fear/Neutral)
- Show trending emotions over time
- Filter posts by detected emotion
- User emotion profile summary
- Leverage existing TimescaleDB continuous aggregates

### 3. Location-Based Sentiment Map (Advanced - ~3-4 hours)
- Interactive map showing sentiment by city/country
- Heat map of emotional trends
- Regional sentiment comparison
- Uses city/country sentiment aggregates (already implemented)

---

## 🤖 Backend/API Features

### Quick Wins (Do These First)

#### 4. Comment/Reply Sentiment Analysis (High Value - ~2 hours)
- Analyze sentiment of comments on posts
- Track emotional tone of conversations
- Identify toxic comment threads
- API: POST /api/comments (auto-analyze on creation)

#### 5. Automated Content Moderation (High Value - ~2 hours)
- Auto-hide/flag posts with HasAnger=true or NeedsReview=true
- Queue system for manual review
- Moderation dashboard API endpoints
- Severity levels (low/medium/high risk)

#### 6. Sentiment Aggregation by Author (Quick - ~30 min)
- Add author-level continuous aggregate (similar to city/country)
- Track user's emotional consistency over time
- API: GET /api/analytics/sentiment/author/{authorKey}

### High Impact (Medium Effort)

#### 7. Real-Time Sentiment Notifications (Medium - ~1.5 hours)
- Alert users when their posts receive angry/negative reactions
- Notify moderators of posts flagged for review (NeedsReview=true)
- SignalR hub for real-time sentiment updates
- Webhook support for external integrations

#### 8. User Sentiment Profile/History (Medium - ~1.5 hours)
- Track user's emotional patterns over time
- API: GET /api/users/{userId}/sentiment-history
- Mental health indicators (e.g., prolonged sadness trends)
- Emotional volatility score

#### 9. Sentiment-Based Recommendations (Complex - ~3 hours)
- "Users who post joyful content" suggestions
- Content filtering: "Show me positive posts only"
- Mood-based feed curation
- API: GET /api/posts/by-sentiment?emotion=Joy&minScore=0.8

### Additional Features

#### 10. Batch Sentiment Re-Analysis (Quick Win - ~1 hour)
- Background job to re-analyze old posts with updated models
- API: POST /api/sentiment/batch-analyze
- Progress tracking and reporting
- Useful when switching from ServerOnly → Adaptive mode

#### 11. Sentiment Webhooks/Events (Medium - ~1.5 hours)
- Publish events when sentiment detected
- Event types: "AngerDetected", "ToxicContentFound", "PositiveViralPost"
- External system integrations
- Azure Event Grid / RabbitMQ / SignalR

#### 12. Multi-Language Sentiment Expansion (Low - ~1 hour)
- Current: English + Spanish keywords
- Expand to: French, Portuguese, Italian, German, etc.
- Leverage existing keyword-based approach

#### 13. Sentiment API Rate Limiting (Low Priority - ~1 hour)
- Prevent abuse of sentiment analysis endpoints
- Quota management per user/API key
- Throttling for expensive operations

#### 14. Sentiment Trend Predictions (Advanced - ~4 hours)
- ML model to predict if sentiment will spike
- "Early warning system" for negative trends
- Time-series forecasting with ML.NET
- API: GET /api/sentiment/predictions

#### 15. Contextual Sentiment (Advanced - ~3 hours)
- Analyze sentiment relative to topic/hashtag
- "People are angry about #politics but happy about #sports"
- Sentiment by business category (products vs services)
- Topic extraction + sentiment correlation
